## 新增需求

### 需求：提交奖品兑换申请
孩子用户必须能够提交奖品的兑换申请。用户必须拥有足够的积分来覆盖奖品费用。

#### 场景：积分充足时提交兑换
- **当** 孩子用户提交可以负担的奖品的兑换
- **则** 系统创建新的 redemption_application 记录，状态为"待审核"，并返回申请 ID

#### 场景：积分不足时提交兑换
- **当** 孩子用户提交无法负担的奖品的兑换
- **则** 系统返回验证错误，提示积分不足

### 需求：查看自己的兑换申请
孩子用户必须能够查看自己的兑换申请历史及状态。

#### 场景：查看兑换历史
- **当** 孩子用户查看自己的兑换申请
- **则** 系统返回所有申请的列表，包含奖品信息、状态和时间戳

### 需求：管理员批准兑换申请
管理员用户必须能够批准或拒绝兑换申请。批准时，积分必须原子性扣除。

#### 场景：批准兑换
- **当** 管理员批准待处理的兑换申请
- **则** 系统更新状态为"已批准"，从用户余额扣除积分，减少奖品库存，并记录管理员备注

#### 场景：拒绝兑换
- **当** 管理员拒绝待处理的兑换申请
- **则** 系统更新状态为"已拒绝"，并记录管理员备注（不扣除积分）

### 需求：查看所有待处理的兑换
管理员用户必须能够查看所有用户的待处理兑换申请。

#### 场景：管理员查看待处理兑换
- **当** 管理员查看待处理的兑换申请
- **则** 系统返回所有状态为"待审核"的申请列表，包含用户和奖品信息
