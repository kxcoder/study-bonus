## 新增需求

### 需求：提交奖励申请
孩子用户必须能够提交奖励申请，说明成就内容。申请必须保存为"待审核"状态。

#### 场景：提交奖励申请
- **当** 孩子用户提交带有描述的奖励申请
- **则** 系统创建新的 reward_application 记录，状态为"待审核"，并返回申请 ID

#### 场景：提交空描述的奖励
- **当** 孩子用户提交空描述的奖励申请
- **则** 系统返回验证错误

### 需求：查看自己的奖励申请
孩子用户必须能够查看自己的奖励申请历史及状态。

#### 场景：查看奖励历史
- **当** 孩子用户查看自己的奖励申请
- **则** 系统返回所有申请的列表，包含描述、状态、积分和时间戳

### 需求：管理员审核奖励申请
管理员用户必须能够审核待处理的奖励申请并批准或拒绝，附带备注。

#### 场景：批准奖励申请
- **当** 管理员批准待处理的奖励申请并设置积分值
- **则** 系统更新申请状态为"已批准"，设置积分值，将积分加到用户余额，并记录管理员备注

#### 场景：拒绝奖励申请
- **当** 管理员拒绝待处理的奖励申请并说明原因
- **则** 系统更新申请状态为"已拒绝"，并记录管理员备注

### 需求：查看所有待处理的奖励申请
管理员用户必须能够查看所有用户的待处理奖励申请。

#### 场景：管理员查看待处理奖励
- **当** 管理员查看待处理的奖励申请
- **则** 系统返回所有状态为"待审核"的申请列表，包含用户信息
