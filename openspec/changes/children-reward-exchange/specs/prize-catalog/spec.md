## 新增需求

### 需求：查看奖品目录
孩子用户必须能够浏览所有可用奖品及其所需积分和剩余库存。

#### 场景：浏览奖品
- **当** 孩子用户查看奖品目录
- **则** 系统返回活跃奖品列表，包含名称、描述、所需积分和剩余库存

#### 场景：按可兑换筛选奖品
- **当** 孩子用户按当前积分余额筛选奖品
- **则** 系统仅返回所需积分不超过用户积分余额的奖品

### 需求：奖品详情
系统必须显示奖品的详细信息，包含名称、描述、所需积分和库存数量。

#### 场景：查看奖品详情
- **当** 孩子用户查看特定奖品
- **则** 系统返回完整的奖品详情，包含所有字段

### 需求：管理员管理奖品
管理员用户必须能够添加、编辑和移除奖品目录中的奖品。

#### 场景：添加新奖品
- **当** 管理员创建新奖品，包含名称、描述、所需积分和库存
- **则** 系统创建新的奖品记录并返回奖品 ID

#### 场景：更新奖品
- **当** 管理员更新奖品详情
- **则** 系统更新奖品记录

#### 场景：移除奖品
- **当** 管理员设置奖品为非活跃
- **则** 系统将奖品标记为 is_active = false 并从目录中隐藏
