## 新增需求

### 需求：孩子用户注册
孩子用户必须能够使用微信账号注册。首次登录时，系统必须创建新的用户记录，角色为"child"，初始积分为0。

#### 场景：首次登录
- **当** 孩子用户首次通过微信登录
- **则** 系统创建新的用户记录，角色为"child"，积分余额为0，并返回成功

#### 场景：再次登录
- **当** 孩子用户使用已有 OpenID 通过微信登录
- **则** 系统返回现有用户记录及其当前积分余额

### 需求：管理员用户认证
管理员用户必须能够使用预注册的凭证登录。系统必须验证用户角色为"admin"后才授予访问权限。

#### 场景：管理员登录
- **当** 管理员用户使用有效凭证登录
- **则** 系统返回角色为"admin"的用户记录，并授予管理员权限

#### 场景：无效管理员凭证
- **当** 管理员用户使用无效凭证登录
- **则** 系统返回认证错误

### 需求：用户角色检查
系统必须提供 API 检查当前用户角色。孩子用户必须无法访问管理员专用 API。

#### 场景：检查用户角色
- **当** 用户查询自身角色
- **则** 系统返回"child"或"admin"
