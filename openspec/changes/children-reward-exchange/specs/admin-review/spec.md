## 新增需求

### 需求：管理员仪表盘概览
管理员用户必须能够查看显示待处理申请数量的仪表盘。

#### 场景：查看仪表盘
- **当** 管理员查看仪表盘
- **则** 系统返回待处理奖励申请和待处理兑换申请的数量

### 需求：筛选申请
管理员用户必须能够按类型（奖励/兑换）、状态和日期范围筛选申请。

#### 场景：按类型筛选
- **当** 管理员按类型"奖励"筛选
- **则** 系统仅返回奖励申请

#### 场景：按状态筛选
- **当** 管理员按状态"待审核"筛选
- **则** 系统仅返回待处理的申请

#### 场景：按日期范围筛选
- **当** 管理员按日期范围筛选
- **则** 系统仅返回该日期范围内创建的申请

### 需求：查看申请详情
管理员用户必须能够查看任何申请的完整详情，包含用户信息和历史。

#### 场景：查看申请详情
- **当** 管理员查看申请详情
- **则** 系统返回所有申请字段、用户信息和管理员备注

### 需求：批量处理申请
管理员用户必须能够一次性批准或拒绝多个申请。

#### 场景：批量批准
- **当** 管理员选择多个待处理申请并批准
- **则** 系统按顺序处理每个申请，更新状态并调整余额
