## 需求背景

原方案设计了两个独立的微信小程序（孩子版和管理员版），维护成本较高。现调整为单一小程序，通过角色区分功能：孩子用户使用奖励/兑换功能，管理员使用审核/管理功能。同时需要预设超级管理员账号，要求首次登录必须修改密码。

## 变更内容

- 调整为单一微信小程序，根据角色动态显示不同功能
- 新增超级管理员账号，默认密码123456
- 首次登录必须强制修改密码
- 根据用户角色显示对应的导航和功能

## 功能模块

### 新增功能

- `role-based-navigation`: 根据用户角色动态显示导航和功能
- `password-change`: 首次登录强制修改密码
- `super-admin-setup`: 预设超级管理员账号

### 修改功能

- `user-auth`: 修改为单一小程序认证，支持角色区分

## 影响范围

- 小程序端需要根据角色动态渲染导航
- 云函数需要根据角色授权不同 API
- 数据库用户表需要新增密码字段
