## 新增需求

### 需求：首次登录强制修改密码
首次登录的管理员必须修改默认密码后才能使用系统功能。

#### 场景：首次登录跳转
- **当** is_first_login 为 true 的用户登录成功
- **则** 系统强制跳转到修改密码页面，禁止访问其他功能

#### 场景：首次登录访问其他页面
- **当** is_first_login 为 true 的用户尝试访问其他页面
- **则** 系统重定向回修改密码页面

### 需求：修改密码
用户必须能够修改自己的密码。

#### 场景：修改密码成功
- **当** 用户输入新密码（至少6位，包含字母和数字）并确认
- **则** 系统更新密码哈希，重置 is_first_login 为 false

#### 场景：密码强度不足
- **当** 用户输入的新密码不满足强度要求
- **则** 系统返回密码强度不足错误

#### 场景：原密码错误
- **当** 用户输入的原密码不正确
- **则** 系统返回原密码错误
