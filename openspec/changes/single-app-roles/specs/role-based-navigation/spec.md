## 新增需求

### 需求：动态角色导航
用户登录后，系统必须根据用户角色显示对应的导航菜单和功能入口。

#### 场景：孩子用户登录
- **当** 角色为 child 的用户登录
- **则** 系统显示：首页、记录奖励、奖励历史、奖品目录、兑换记录

#### 场景：管理员登录
- **当** 角色为 admin 或 super_admin 的用户登录
- **则** 系统显示：仪表盘、奖励审核、兑换审核、奖品管理

### 需求：功能权限控制
不同角色只能访问对应的功能 API。

#### 场景：孩子访问管理功能
- **当** 孩子用户尝试访问管理功能 API
- **则** 系统返回权限不足错误

#### 场景：管理员访问兑换功能
- **当** 管理员尝试访问兑换相关 API
- **则** 系统返回权限不足错误
